**Инфра-находки при подключении фронт↔бэк через rewrite (next.config.ts):**
- Next.js 16 сам 308-редиректит URL с завершающим слэшем на URL без него (до применения rewrite) — конфликтует с Django `APPEND_SLASH`, который для не-GET запросов вместо редиректа кидает `RuntimeError`. Решение: `skipTrailingSlashRedirect: true` в `next.config.ts`.
- Даже с этим флагом, catch-all `:path*` в rewrite destination теряет завершающий слэш при реконструкции пути — решение: явный литеральный `/` в конце destination (`` `${internalApiUrl}/api/:path*/` ``).
- `DJANGO_ALLOWED_HOSTS` должен включать `backend` (имя docker-сервиса) — Next-прокси шлёт Host-заголовок с адресом назначения, Django иначе кидает `DisallowedHost`. Добавлено в `.env` и `.env.example`.
- Изменения `environment:` в `docker-compose.yml` не подхватываются через `docker compose restart <service>` — нужен `docker compose up -d <service>` (пересоздание контейнера), иначе новая переменная окружения не появится внутри контейнера.
- **CSRF 403 на регистрации/логине из браузера (найдено и починено 2026-07-01).** Причина: если в том же браузере залогинены в Django Admin (`/admin/`, порт 8000), Django-сессия ставится без `Domain` → cookie скоуплен на хост `localhost` **независимо от порта** → браузер несёт эту `sessionid` и на запросы к `:3000/api/...`. `SessionAuthentication` (был в `DEFAULT_AUTHENTICATION_CLASSES` по умолчанию) видел валидную сессию и включал `enforce_csrf()`, а наш JWT-фронтенд никогда не шлёт ни `Origin` из доверенных, ни CSRF-токен. `curl` без cookie-jar никогда не задевал эту ветку — поэтому баг не проявлялся в ручных curl-проверках. Одного `CSRF_TRUSTED_ORIGINS` **недостаточно** (закрывает только Origin-проверку, следом падает отдельная проверка CSRF-токена, который фронт не отправляет и не должен). Полный фикс — убрать `SessionAuthentication` из `DEFAULT_AUTHENTICATION_CLASSES` (API — только JWT, Django Admin работает отдельно и в этой настройке не участвует), `CSRF_TRUSTED_ORIGINS` оставлен как defense-in-depth. Оба фикса в `backend/config/settings.py`, воспроизведено и подтверждено через curl с реальной Django-сессией (cookie jar) до и после фикса.

  (`/my-bids/` + пересечение на клиенте), а аннотацией `Exists(Bid.objects.filter(
  request=OuterRef("pk"), contractor=user))` прямо в `get_queryset()`
