# Сессия 2026-07-01 — Ребрендинг EOSpatial → ПроГео

## Решение сессии
Проект получил название **«ПроГео»** (латиницей: **ProGeo**). Заменено плейсхолдер-название
«EOSpatial» / «EOSpatial Marketplace» во всех бренд-видимых местах фронтенда, бэкенда и
документации.

## Что было сделано

### Замена бренд-текста
- `frontend/src/components/ui/AppNav.tsx` — логотип в шапке
- `frontend/src/components/ui/AppFooter.tsx` — логотип, копирайт, email (`support@progeo.kz`)
- `frontend/src/app/[locale]/layout.tsx` — `<title>` / metadata
- `frontend/src/app/[locale]/landing/Screen.tsx` — лого шапки, заголовок секции «Почему ПроГео»,
  текст про модератора, лого футера, копирайт **и** email в собственном инлайновом футере
  лендинга (пропустили в первый проход — у лендинга своя копия футера отдельно от `AppFooter`,
  почему это техдолг — см. ниже)
- `backend/config/settings.py` — заголовок Swagger/OpenAPI (`SPECTACULAR_SETTINGS.TITLE`)
- `docs/design-system.md` — заголовок файла

### Удалены устаревшие артефакты
- `/ru/institutional` (`page.tsx` + `Screen.tsx`) — статичный демо-референс первого рабочего
  экрана без реального API; решили не переименовывать, а удалить, т.к. реальная лента заявок
  будет строиться заново по дизайн-системе. Ссылку «Лента заявок» в `AppNav.tsx` временно
  указали на `#`.
- `design-system/eospatial/MASTER.md` (и папка `design-system/eospatial/`) — черновой артефакт
  раннего дизайн-эксперимента с несовпадающей палитрой (фиолетовый/зелёный), не соответствующий
  выбранному институциональному стилю. Проверили — ничего в коде на него не ссылалось. Единственный
  источник правды по дизайну — `docs/design-system.md`.

### Инфраструктурные дефолты
- `.env.example` и `backend/config/settings.py` — дефолты `POSTGRES_DB`/`POSTGRES_USER`/
  `POSTGRES_PASSWORD`/`MINIO_BUCKET` обновлены `eospatial` → `progeo`; комментарий-пример
  домена `files.eospatial.kz` → `files.progeo.kz`.
- Реальный локальный `.env` и существующие Docker volume (Postgres, MinIO) **намеренно не
  тронуты** — чтобы не терять локальные dev-данные. Смена потребует пересоздания volume,
  зафиксировано как техдолг на момент следующего пересоздания dev-среды или первого
  прод-деплоя.

### Что не трогали (сознательно)
- `docs/progress.md` — путь `D:\EOSpatial\2026\Startup\Project` (реальный путь к старой копии
  на диске, историческая запись, не бренд)
- `docs/sessions/2026-06-22-*.md`, `2026-06-24-*.md`, `2026-06-25-*.md` — реальные email/имена
  бакетов, существовавшие на момент тех сессий (исторические факты)
- `frontend/package.json` — `name: "frontend"`, уже нейтральное, не бренд-плейсхолдер

### Проверка после изменений
- Финальный регистронезависимый grep по всему репозиторию — «eospatial» остался только в
  согласованных исключениях (историческая запись).
- Backend перезапущен: логи чистые, `/admin/` → 302 (ожидаемо), `/api/schema/` → title
  «ПроГео API».
- Frontend перезапущен: `/ru/landing` → 200, `/ru` → 200, `/ru/institutional` → 404
  (подтверждает удаление); 7 упоминаний «ПроГео» на странице, 0 — старого названия; то же
  самое на `/kk/landing` и `/en/landing` (контент лендинга пока не локализован по языкам —
  открытый пункт 1.8 i18n).

1. Инфраструктура `eospatial` → `progeo`: дефолты обновлены, реальная среда — нет (см. выше).
2. `/ru/landing` (`Screen.tsx`) верстает свой инлайновый футер вместо общего `AppFooter.tsx` —

   лендинга: заменить на `<AppFooter />`.

2. **Маршрут лендинга:** `/ru/landing` vs корень locale (`/ru`).
