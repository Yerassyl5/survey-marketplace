### Инфраструктурные фиксы MinIO (найдены при тестировании 1.2)
- **Авто-создание бакета**: бакет `eospatial` не создавался сам при первом старте MinIO (в отличие от Postgres с `init.sql`) — без него любая загрузка падала с `NoSuchBucket`. Добавлен одноразовый сервис `minio-init` (`minio/mc`) в `docker-compose.yml`: ждёт здорового `minio` (добавлен healthcheck), делает `mc mb --ignore-existing`. `backend`/`celery-worker`/`celery-beat` стартуют только после `minio-init` (`condition: service_completed_successfully`). Проверено на полностью пересозданном volume `minio_data` — бакет появляется сам, идемпотентно.
- **Публичные ссылки на файлы**: presigned-ссылки указывали на внутренний docker-хост `minio:9000`, недоступный браузеру снаружи. Добавлен `backend/common/storage.py` (`PublicURLS3Storage`) — операции с файлами идут через внутренний `MINIO_ENDPOINT`, а ссылки подписываются отдельным boto3-клиентом под `MINIO_PUBLIC_ENDPOINT` (dev: `http://localhost:9000`, в проде — заменить на реальный домен MinIO в `.env`). Подпись валидна, потому что вычисляется локально под целевой хост — реальное сетевое соединение с этим адресом для подписи не нужно.

### Permissions (.claude/settings.local.json)
По запросу пользователя сужен allow-list:
- `Bash(docker compose *)` → отдельные узкие правила (`up`, `ps`, `logs`, `exec`, `restart`, `build`); `down`, `rm`, `volume` теперь вне allow-list — требуют подтверждения.
- `Bash(git push *)` убран из allow-list — push всегда спрашивает подтверждение.
- `rm -rf node_modules package.json package-lock.json` оставлен как есть (уже фиксированный путь, не wildcard).
