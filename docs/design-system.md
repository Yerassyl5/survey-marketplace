# ПроГео — Дизайн-система (Институциональный стиль)

> **Читай этот файл в начале каждой сессии при работе с фронтендом.**
> Все новые экраны строятся только в этом стиле.

---

## Выбор и обоснование

**Выбрано: Направление 1 «Институциональный»** (из трёх протестированных вариантов, июль 2026).

Причина: оптимален для B2B ежедневного использования — чистый, читаемый, профессиональный, вызывает доверие у серьёзной аудитории (проектные институты, застройщики, госструктуры Казахстана).

---

## Источник истины: CSS-переменные

Все цвета, отступы, шрифты — в `frontend/src/app/globals.css` (`--ds-*` namespace).
**Менять токены только там.** Компоненты ссылаются на переменные, не хардкодят значения.

Метки соответствуют реальному жизненному циклу (`marketplace.RequestStatus`:
open → awarded → result_submitted → accepted) — уточнено при подключении
`StatusBadge` к реальным данным в «Мои заявки» (2026-07-07); исходный эскиз
дизайн-системы использовал другие названия до этого коммита нигде не применялся.

| Статус | Backend-статус | Фон-токен | Текст-токен |
|---|---|---|---|

## Тёмная тема

Включается добавлением класса `dark` на `<html>`. Все `--ds-*` токены и shadcn-переменные
переопределяются автоматически через CSS `.dark { ... }` блок в `globals.css`.

Принцип тёмной темы:
- `--ds-navy` → `#020617` (почти чёрный)
- `--ds-bg-white` → `#0F172A` (карточки)
- `--ds-blue` → `#38BDF8` (светло-голубой акцент)
- Статус/верификационные цвета — мягче, под тёмный фон

## Правила применения

1. **Новый экран = AppNav + контент + AppFooter** — не верстать шапку/подвал с нуля
2. **Цвета только через CSS-переменные** — не хардкодить `#0369A1` в компонентах
3. **Шрифты не импортировать в page.tsx** — они уже в layout.tsx
4. **Статусы и верификацию** — всегда через `StatusBadge` / `VerificationBadge`
5. **Пагинация** — всегда через `Pagination`, не самодельная
6. **Focus-кольца** — не убирать `outline` без замены; `:focus-visible` настроен глобально
7. **cursor-pointer** — на всех кликабельных элементах
8. **Transitions** — 150ms для hover, не длиннее 300ms
9. **Новый роут** на Windows+Docker — всегда `docker compose restart frontend`
